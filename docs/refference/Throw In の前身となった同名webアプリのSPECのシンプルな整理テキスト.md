# Throw In 画面・機能整理

## 0. 整理方針

この整理では、**SPEC(3).md に明示されている主要4画面**を中心にまとめています。
色味、文字サイズ、表示幅、秒数などの定量情報は除外しています。

根拠資料は、主に `SPEC(3).md`、補助として `01_upstream_spec(3).md` と `PROJECT(3).md` を参照しています。  

※SPEC(3).md : Throw In の前身となった同名webアプリのSPEC書

---

## 1. プロダクト全体の本質

**Throw In の本質は、情報を先に受け止め、あとから整理・検索できる個人ライブラリを作ることです。**

```text
URL / 画像 / PDF / 日記
  → まず保存する
    → UserClip を作る
      → AI が非同期で整理する
        → 一覧・詳細・検索であとから取り出す
```

重要な考え方は以下です。

| 観点     | 本質                                  |
| ------ | ----------------------------------- |
| 保存     | **AI 完了を待たず、UserClip 作成時点で成功扱いにする** |
| 整理     | **AI 処理は裏側で進める**                    |
| 再訪     | **一覧・詳細・検索で、あとから見つけ直せるようにする**       |
| 手動情報   | **手動タイトルやメモを AI より優先する**            |
| プライバシー | **個人データを公開面に直接出さない**                |

---

# 2. 主要画面一覧

SPEC(3).md で具体化されている主要画面は、以下の4つです。

| 画面          |          ルート | 主な役割             | 本質                |
| ----------- | -----------: | ---------------- | ----------------- |
| Library     |          `/` | 保存済みクリップの一覧・管理   | **再訪の入口**         |
| Add Clip    |       `/add` | 新規保存             | **情報を受け止める入口**    |
| Clip Detail | `/clip/[id]` | 個別クリップの詳細閲覧・状態管理 | **1件の価値を確認する場所**  |
| Search      |    `/search` | クリップ検索           | **埋もれた情報を取り出す場所** |

---

# 3. 画面ごとの整理

---

## 3.1 Library 画面

### 目的意識

**保存したクリップを一覧で見て、あとから再訪できるようにする画面です。**

ユーザーはこの画面で、保存済み情報の状態を確認し、必要なクリップを開きます。
また、新規保存や検索への起点にもなります。

---

### 画面構成と本質

| 構成要素    | 役割               | 本質               |
| ------- | ---------------- | ---------------- |
| ヘッダー    | アプリ名表示、検索開始      | ライブラリの入口であることを示す |
| インライン検索 | 一覧画面内で検索する       | 画面を離れずに探せる       |
| クリップ一覧  | 保存済み情報を並べる       | 再訪対象を選ぶ          |
| クリップカード | タイトル、種別、状態、要約を表示 | そのクリップを開くべきか判断する |
| FAB     | Add Clip へ移動     | 保存導線を常に近くに置く     |

```text
Library
  ├─ ヘッダー
  │   └─ 検索開始
  ├─ インライン検索
  ├─ クリップ一覧
  │   └─ クリップカード
  └─ Add Clip への導線
```

---

### 機能一覧

| 機能            | 内容                        |
| ------------- | ------------------------- |
| クリップ一覧表示      | 保存済みクリップを時系列で表示する         |
| 無限スクロール       | クリップを追加読み込みする             |
| インライン検索       | Library 画面内で検索欄を展開する      |
| 全文検索          | タイトル、URL、メモ、要約などを横断検索する   |
| 既読 / 未読表示     | クリップの読了状態を識別する            |
| コンテンツ種別表示     | 記事、画像、PDF、日記を区別する         |
| 処理状態表示        | 処理待ち、処理中、部分抽出などを表示する      |
| Stage2 選択済み表示 | 詳細AI分析対象のクリップを示す          |
| 詳細遷移          | カードタップで Clip Detail へ移動する |
| 新規保存導線        | FAB から Add Clip へ移動する     |

---

### キャプチャ関連

Library 画面そのものでは、直接キャプチャは行いません。
ただし、**Add Clip への入口**と、**キャプチャ後の状態確認場所**として機能します。

| 関与        | 内容                                   |
| --------- | ------------------------------------ |
| 保存開始      | FAB から Add Clip へ移動する                |
| 保存後表示     | 新規作成されたクリップを一覧に表示する                  |
| 非同期処理の可視化 | pending / ready / partial などの状態を表示する |
| 再訪導線      | 保存した情報をあとから開けるようにする                  |

## 3.2 Add Clip 画面

### 目的意識

**URL、画像、PDF、日記を新しく保存する画面です。**

この画面の本質は、情報をきれいに整理することではありません。
**まず失わずに保存すること**です。

---

### 画面構成と本質

| 構成要素       | 役割                      | 本質               |
| ---------- | ----------------------- | ---------------- |
| ヘッダー       | 保存画面であることを示す            | 行動を明確にする         |
| モードセレクター   | URL / 画像・PDF / 日記を切り替える | 入力種別を選ばせる        |
| URL モード    | URL を保存する               | Web 情報を受け止める     |
| Upload モード | 画像・PDF を保存する            | ファイル情報を受け止める     |
| 日記モード      | 本文と日付を保存する              | 自分の文脈を残す         |
| 任意メタ情報     | タイトルやメモを入力する            | 後から探しやすくする       |
| 保存ボタン      | API を呼び出す               | UserClip 作成を開始する |
| 成功通知       | 保存成功を知らせる               | AI完了を待たずに安心させる   |
| エラー表示      | 入力不備や保存失敗を知らせる          | 保存できない理由を明確にする   |

```text
Add Clip
  ├─ URL モード
  │   ├─ URL
  │   ├─ 手動タイトル
  │   └─ メモ
  ├─ Upload モード
  │   ├─ 画像 / PDF 選択
  │   ├─ 手動タイトル
  │   └─ メモ
  └─ 日記モード
      ├─ 本文
      ├─ 日付
      ├─ タイトル
      └─ 写真添付
```

---

### 機能一覧

| 機能               | 内容                       |
| ---------------- | ------------------------ |
| モード切替            | URL、画像・PDF、日記の入力形式を切り替える |
| URL 保存           | URL を入力してクリップを作成する       |
| URL 検証           | URL 形式や危険なアクセス先を検証する     |
| 画像アップロード         | 画像ファイルを保存する              |
| PDF アップロード       | PDF ファイルを保存する            |
| 日記作成             | 本文、日付、任意タイトルを保存する        |
| 手動タイトル入力         | AIより優先されるタイトルを保存する       |
| メモ入力             | ユーザーの補足情報を保存する           |
| アップロード準備         | Presigned URL を取得する      |
| 直接アップロード         | ブラウザから Storage へアップロードする |
| アップロード完了後のクリップ作成 | アップロード済みファイルをクリップ化する     |
| 入力バリデーション        | 入力内容を保存前に検証する            |
| 保存成功通知           | 保存完了を通知する                |
| Home 遷移          | 保存後に Library へ戻る         |
| エラーハンドリング        | エラー内容を表示する               |

---

### キャプチャ関連

Add Clip は、**キャプチャ機能の中心画面**です。

| キャプチャ種別 | 入力                | 作成されるもの                      | 後続処理            |
| ------- | ----------------- | ---------------------------- | --------------- |
| URL     | URL、任意タイトル、メモ     | UserClip、Content、Job         | URL取得、本文抽出、AI整理 |
| 画像      | 画像ファイル、任意タイトル、メモ  | UploadSession、Asset、UserClip | 保存、必要に応じて解析     |
| PDF     | PDFファイル、任意タイトル、メモ | UploadSession、Asset、UserClip | テキスト抽出、部分抽出判定   |
| 日記      | 本文、日付、任意タイトル、写真   | UserClip、必要に応じてAsset         | ライブラリ化、検索対象化    |
| 手動情報    | タイトル、メモ           | UserClip の manual/user_note  | 表示・検索に利用        |

## 3.3 Clip Detail 画面

### 目的意識

**1つのクリップを詳しく見て、状態を管理する画面です。**

一覧では判断できない内容を確認し、既読化や Stage2 選択などの操作を行います。
この画面の本質は、**保存した情報を意味ある1件として理解すること**です。

---

### 画面構成と本質

| 構成要素         | 役割              | 本質           |
| ------------ | --------------- | ------------ |
| タイトル         | クリップの主題を表示する    | 何の情報かを即座に伝える |
| メタ情報         | ドメイン、日付、種別を表示する | 出どころと保存文脈を示す |
| 処理状態         | AI処理や抽出状態を表示する  | 未完成状態も隠さない   |
| 既読 / 未読操作    | 読了状態を切り替える      | 自分の利用状態を残す   |
| Stage2 表示・操作 | 詳細AI分析対象を管理する   | 深く整理する対象を選ぶ  |
| AI要約         | AI生成の短い説明を表示する  | 内容把握を助ける     |
| AI本文         | 抽出・生成された本文を表示する | 詳細理解を助ける     |
| AIタグ         | 分類・検索補助を表示する    | 後から探しやすくする   |
| メモ           | ユーザー自身の補足を表示する  | 個人の文脈を残す     |
| 元URL         | 元記事を開く          | 原文へ戻れるようにする  |

```text
Clip Detail
  ├─ タイトル・メタ情報
  ├─ 状態表示
  ├─ 既読 / 未読
  ├─ Stage2 操作
  ├─ AI 要約
  ├─ AI 本文
  ├─ AI タグ
  ├─ ユーザーメモ
  └─ 元URL
```

---

### 機能一覧

| 機能             | 内容                                             |
| -------------- | ---------------------------------------------- |
| 詳細取得           | 1件のクリップ情報を取得する                                 |
| タイトル表示         | manual_title → ai_title → source_title の順で表示する |
| メタ情報表示         | ドメイン、保存日、コンテンツ種別を表示する                          |
| 処理状態表示         | ready / partial / pending などの状態を表示する           |
| AI表示制御         | ai_visibility に応じてAI情報の表示範囲を制御する               |
| AI要約表示         | AIが生成した要約を表示する                                 |
| AI本文表示         | AI本文を表示する                                      |
| AIタグ表示         | AIタグやカテゴリを表示する                                 |
| 既読 / 未読切替      | クリップの読了状態を更新する                                 |
| Stage2 選択 / 解除 | 詳細AI分析対象にするか切り替える                              |
| クォータ確認         | Stage2 の残数や利用状況を表示する                           |
| メモ表示           | ユーザーが入力したメモを表示する                               |
| 元URLを開く        | source_url がある場合に原文へ移動する                       |
| AIバッジ表示        | AI生成情報であることを示す                                 |

---

### キャプチャ関連

Clip Detail は、キャプチャを直接行う画面ではありません。
ただし、**キャプチャ後の価値確認と状態管理**を担います。

| 関与         | 内容                       |
| ---------- | ------------------------ |
| 保存結果の確認    | 保存された内容を詳細表示する           |
| AI処理結果の確認  | 要約、本文、タグを確認する            |
| 不完全状態の確認   | partial や pending を確認できる |
| Stage2 対象化 | 保存後に深い整理対象として選択する        |
| 原文再訪       | 元URLへ戻る                  |

## 3.4 Search 画面

### 目的意識

**保存したクリップを全文検索で探す画面です。**

SPEC上では、Library 画面内のインライン検索が主導線です。
Search 画面は、独立した検索ページまたはディープリンク用途として位置づけられています。

---

### 画面構成と本質

| 構成要素   | 役割             | 本質          |
| ------ | -------------- | ----------- |
| 検索バー   | 検索語を入力する       | 探したい情報を指定する |
| 検索ボタン  | 明示的に検索を実行する    | 操作を確定する     |
| 結果一覧   | ヒットしたクリップを表示する | 候補から再訪対象を選ぶ |
| 空状態    | 検索前の案内を出す      | 何をすればよいか示す  |
| 読み込み状態 | 検索中であることを示す    | 待機状態を明確にする  |
| エラー状態  | 検索失敗を知らせる      | 再試行できるようにする |

```text
Search
  ├─ 検索バー
  ├─ 検索ボタン
  ├─ 状態表示
  │   ├─ Empty
  │   ├─ Pending
  │   ├─ Error
  │   └─ Results
  └─ 検索結果一覧
```

---

### 機能一覧

| 機能      | 内容                       |
| ------- | ------------------------ |
| キーワード検索 | 入力語でクリップを検索する            |
| 遅延検索    | 入力後に少し待って検索を実行する         |
| 明示検索    | 検索ボタンで検索を実行する            |

| 読み込み表示  | 検索中の状態を表示する              |

### キャプチャ関連

役割は **保存済み情報の回収** です。

| 関与         | 内容                  |
| ---------- | ------------------- |
| 直接保存       | なし                  |
| 保存済み情報の検索  | あり                  |
| キャプチャ後の再利用 | あり                  |
| 詳細再訪       | 検索結果から Detail へ移動する |

# 4. キャプチャ機能一覧

SPEC(3).md のキャプチャ機能を、実装観点ではなく画面理解しやすい形に整理します。

| ID     | 機能                   | 内容                       | 主な画面               |
| ------ | -------------------- | ------------------------ | ------------------ |
| CAP-01 | URL 保存               | URLを入力して保存する             | Add Clip           |
| CAP-02 | SSRF ガード             | 危険なURLや内部ネットワークへのアクセスを防ぐ | Add Clip / Backend |
| CAP-03 | 画像アップロード             | 画像ファイルを保存する              | Add Clip           |
| CAP-04 | PDFアップロード            | PDFファイルを保存する             | Add Clip           |
| CAP-05 | 日記作成                 | 日付と本文を持つ日記クリップを作成する      | Add Clip           |
| CAP-06 | 手動タイトル / メモ          | 任意のタイトルや補足メモを保存する        | Add Clip / Detail  |
| CAP-07 | Presigned URL アップロード | ブラウザからStorageへ直接アップロードする | Add Clip / Backend |
| CAP-08 | 保存成功通知               | 保存完了を通知し、Libraryへ戻す      | Add Clip / Library |

---

## 4.1 キャプチャ種別ごとの本質

| 種別   | 目的             | 本質                    |
| ---- | -------------- | --------------------- |
| URL  | Web記事やページを保存する | **あとで読める情報を逃さない**     |
| 画像   | 画像情報を保存する      | **視覚情報をライブラリに入れる**    |
| PDF  | PDF資料を保存する     | **資料を検索・再訪対象にする**     |
| 日記   | 自分の記録を保存する     | **外部情報だけでなく自分の文脈も残す** |
| 手動情報 | タイトルやメモを付ける    | **AI任せにせず、自分の意図を残す**  |

---

## 4.2 キャプチャ後の流れ

```text
入力する
  → バリデーションする
    → UserClip を作る
      → 保存成功を通知する
        → Library に戻る
          → AI処理は裏側で進む
            → 状態が一覧・詳細に反映される
```

重要なのは、**保存成功とAI処理完了を分けること**です。
これにより、ユーザーはAI処理を待たずに次の行動へ移れます。

# 6. 上流資料にある周辺・予定画面

以下は `01_upstream_spec(3).md` や `PROJECT(3).md` に含まれる画面です。
ただし、SPEC(3).md の現在のナビゲーションでは、主要画面としては具体化されていません。

| 画面                      | 状態         | 目的                   | 本質                    |
| ----------------------- | ---------- | -------------------- | --------------------- |
| Auth / 初期導線             | 周辺機能       | サインイン・サインアップ・メール確認   | **個人ライブラリを守る入口**      |
| Mobile Share Intake     | 要注意        | OS共有シートからURLや画像を受け取る | **外部から保存へ流し込む入口**     |
| Edit Metadata           | 上流仕様       | 手動タイトル・タグ・メモを編集する    | **AIではなく人の意図を反映する場所** |
| Manual Reports          | 上流仕様       | 日・週・月・年単位で振り返る       | **蓄積を見返す場所**          |
| Settings / Plan         | 上流仕様       | プラン・権利・クォータを確認する     | **利用権限を理解する場所**       |
| Admin Dashboard         | Phase 5 予定 | 運用状態を確認する            | **管理者の司令塔**           |
| Admin PublicCard Review | Phase 5 予定 | 公開候補カードを審査する         | **公開面の安全確認場所**        |
| Admin Jobs & Policies   | Phase 5 予定 | ジョブ再処理やAI方針を管理する     | **裏側の処理を制御する場所**      |

補足です。
Mobile Share Intake は上流資料では重要導線として扱われていますが、SPEC(3).md では PWA 中心かつ Share Extension は非対応として整理されています。したがって、現時点では **主要実装画面ではなく、将来候補または仕様差分のある項目**として扱うのが安全です。

| --------------- | -------------- | ---------------------- |
| Library         | 保存済みクリップを見る    | **保存した情報の再訪ハブ**        |
| Add Clip        | 新しい情報を保存する     | **情報を逃さない入口**          |
| Clip Detail     | 1件の内容を確認・管理する  | **保存した情報を意味ある単位にする場所** |
| Search          | 保存済み情報を探す      | **埋もれた情報を取り出す場所**      |
| Auth            | 利用者を確認する       | **個人ライブラリを守る入口**       |
| Settings / Plan | 権利やクォータを見る     | **利用可能な整理能力を確認する場所**   |
| Admin           | 運用・公開・ジョブを管理する | **安全性と継続運用を支える場所**     |

# 8. 最終整理

Throw In の画面設計は、以下の4つに分けると理解しやすいです。

```text
1. 保存する
   → Add Clip

2. 見る
   → Library

3. 理解する・管理する
   → Clip Detail

4. 探す
   → Search
```

つまり、画面全体の流れはこうです。

```text
Add Clip
  → 情報を保存する

Library
  → 保存した情報を一覧する

Clip Detail
  → 1件の内容を確認する

Search
  → 必要な情報を探し直す
```

最も重要な設計思想は、**保存を止めないこと**です。
AI処理、要約、タグ付け、Stage2分析はその後に続く補助処理です。
画面ごとの役割も、この思想に沿って整理できます。
